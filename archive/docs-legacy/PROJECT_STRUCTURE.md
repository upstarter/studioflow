# StudioFlow Project Structure Guide

## Overview
This guide provides comprehensive documentation on the StudioFlow project directory structure, explaining what goes where, why, and how each directory integrates with the StudioFlow tools and general video production workflows.

## Project Naming Convention
```
YYYYMMDD_Project_Name/
```
- **Date prefix**: Automatic chronological sorting
- **Underscores**: Replace spaces for compatibility
- **Example**: `20250114_Python_Tutorial_Advanced`

## StudioFlow Tool Integration

### sf-project Integration
```bash
# Creates the entire structure
sf-project create "My Video" --template youtube

# Automatically:
# - Creates dated directory
# - Sets up all folders
# - Initializes .sf/project.json
# - Creates README.md
# - Initializes git repo
```

### sf-obs Integration
```bash
sf-obs setup "My Video"

# Automatically configures:
# - Recording path → 01_FOOTAGE/OBS_RECORDINGS/
# - Replay buffer → 01_FOOTAGE/HIGHLIGHTS/
# - Scene collection → .sf/obs_config.json
```

### sf-capture Integration
```bash
sf-capture snap
# Saves to → 01_FOOTAGE/SCREENSHOTS/

sf-capture record
# Saves to → 01_FOOTAGE/SCREEN_RECORDINGS/

sf-capture grid *.png
# Creates → 01_FOOTAGE/SCREENSHOTS/comparison_grid.png
```

### sf-audio Integration
```bash
sf-audio transcribe video.mp4
# Creates → 02_AUDIO/TRANSCRIPTS/video.srt

sf-audio denoise voice.wav
# Creates → 02_AUDIO/VO/processed/voice_clean.wav

sf-audio library search "upbeat"
# Copies to → 02_AUDIO/MUSIC/
```

### sf-youtube Integration
```bash
sf-youtube metadata "My Video"
# Creates → 07_DOCS/METADATA/youtube_metadata.json

sf-youtube thumbnail-test "My Video"
# Sets up → 03_GRAPHICS/THUMBNAILS/variant_[A|B|C]/

sf-youtube hooks "My Video"
# Saves to → 07_DOCS/SCRIPT/hooks.json
```

### sf-ai Integration
```bash
sf-ai script "My Video"
# Creates → 07_DOCS/SCRIPT/script.md

sf-ai ideas "topic"
# Saves to → 07_DOCS/NOTES/ideas.md
```

### sf-resolve Integration
```bash
sf-resolve setup "My Video"
# Creates → 04_EDIT/TIMELINE/project.drp

sf-resolve import "My Video"
# Imports from all numbered directories

sf-resolve render "My Video"
# Exports to → 05_EXPORT/FINAL/
```

### sf-storage Integration
```bash
sf-storage status
# Shows usage of all projects in /mnt/studio/Projects/

sf-storage archive "My Video"
# Moves entire project to /mnt/archive/

sf-storage move "My Video" render
# Moves to /mnt/render/ for export processing
```

## Workflow Examples

### Complete YouTube Workflow
```bash
# 1. Setup
PROJECT="Python_Tutorial"
sf-project create "$PROJECT" --template youtube

# 2. Pre-production
sf-ai script "$PROJECT" --style educational
sf-youtube titles "Python Programming" --style tutorial

# 3. Production
sf-obs setup "$PROJECT"
# Record in OBS... files go to 01_FOOTAGE/OBS_RECORDINGS/

# 4. Capture B-roll
sf-capture snap --window  # → 01_FOOTAGE/SCREENSHOTS/
sf-capture grid *.png      # → comparison_grid.png

# 5. Audio processing
sf-audio transcribe "$PROJECT/01_FOOTAGE/OBS_RECORDINGS/*.mkv"
# Creates → 02_AUDIO/TRANSCRIPTS/

# 6. Editing
sf-resolve setup "$PROJECT"
sf-resolve import "$PROJECT"  # Imports all media
sf-resolve color "$PROJECT" --style viral

# 7. Export
sf-resolve render "$PROJECT" --preset youtube
# → 05_EXPORT/FINAL/YouTube_1080p.mp4

# 8. Platform optimization
sf-youtube metadata "$PROJECT"
sf-youtube thumbnail-test "$PROJECT"

# 9. Archive
sf-storage archive "$PROJECT"
```

### Quick Instagram Reel
```bash
# 1. Quick setup
sf-project create "Quick_Tip" --template minimal

# 2. Record
sf-capture record --duration 30 --vertical
# → FOOTAGE/recording.mp4

# 3. Edit & Export
sf-resolve render "Quick_Tip" --preset instagram
# → EXPORT/instagram_reel.mp4

# 4. Metadata
sf-youtube metadata "Quick_Tip" --platform instagram
```

## Best Practices

### File Naming
```
Purpose_Version_Date.ext

Examples:
main_narration_v2_20250114.wav
thumbnail_variantA_final.png
script_draft_v3.md
```

### Version Control
```bash
# .gitignore (auto-created)
04_EDIT/CACHE/
*.cache
*.tmp
05_EXPORT/DRAFTS/
```

### Organization Tips
1. **Keep source files in numbered directories** (01-07)
2. **Use subfolders for organization** (A_ROLL, B_ROLL)
3. **Name files descriptively** (what_version_date)
4. **Archive completed projects** to free active storage
5. **Use .sf/ for tool metadata** (never edit manually)

### Storage Management
```bash
# Check project size
sf-storage status

# When project is complete
sf-storage archive "Project_Name"

# For active editing
sf-storage move "Project_Name" active

# For rendering
sf-storage move "Project_Name" render
```

## Platform-Specific Considerations

### YouTube
- Keep master at highest quality (4K if possible)
- Multiple thumbnail variants for A/B testing
- Separate SHORTS directory for vertical content
- Detailed metadata for SEO

### Instagram
- Vertical format (1080x1920)
- 60-second maximum for reels
- Square option (1080x1080) for feed
- Compressed audio for mobile

### TikTok
- Vertical only (1080x1920)
- 15-60 second clips
- Trending audio in 02_AUDIO/MUSIC/
- Fast-paced edit style

## Automation Opportunities

### Auto-Import Script
```bash
#!/bin/bash
# Auto-import from SD card to project
PROJECT="$1"
sf-capture import /media/sdcard/ "$PROJECT/01_FOOTAGE/A_ROLL/"
sf-audio process "$PROJECT/01_FOOTAGE/A_ROLL/*.mp4"
```

### Batch Processing
```bash
# Process all screenshots
for img in 01_FOOTAGE/SCREENSHOTS/*.png; do
  sf-capture standardize "$img"
done

# Transcribe all videos
for video in 01_FOOTAGE/*/*.mp4; do
  sf-audio transcribe "$video"
done
```

### Project Templates
Create custom templates in `~/.config/studioflow/templates/` for specific content types (gaming, vlog, podcast, etc.)

## Troubleshooting

### Common Issues
1. **"No space in active tier"** - Use `sf-storage archive` old projects
2. **"Can't find footage"** - Check correct numbered directory
3. **"Resolve can't import"** - Ensure files are in standard directories
4. **"OBS not recording to project"** - Run `sf-obs setup` first

### Directory Permissions
```bash
# Fix permissions if needed
chmod -R 755 ~/Projects/20250114_My_Video/
```

### Finding Files
```bash
# Search across project
find . -name "*.mp4" -type f

# List by modification time
ls -lt 01_FOOTAGE/**/*
```

## Summary

The StudioFlow project structure is designed to:
- **Organize automatically** - Tools know where to put files
- **Scale with complexity** - From minimal to full production
- **Support collaboration** - Clear structure anyone can follow
- **Enable automation** - Consistent paths for scripts
- **Optimize workflow** - Everything has its place

By following this structure, you ensure compatibility with all StudioFlow tools and maintain a professional, organized workflow from capture to final export.
